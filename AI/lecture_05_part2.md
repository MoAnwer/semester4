### Types of Search Algorithms

- **Uninformed Search Strategies:**
    - These algorithms only use information provided by the problem definition: initial state, available actions, transition model, goal state, and solution path cost.
    - Examples: Depth-First Search (DFS), Breadth-First Search (BFS), Depth-Limited Search (DLS), Iterative Deepening Search (IDS), Uniform-Cost Search (UCS).
- **Informed Search Strategies:**
    - These algorithms use additional information to estimate the "promise" of an action or state, specifically the estimated cost from a state to a goal.
    - They utilize an evaluation function, f(n)f(n), to prioritize nodes for exploration.
    - Examples: Greedy Best-First Search (GBFS), A* Search.

### Tree Search Algorithm Outline

The general outline for tree search algorithms is as follows:

1. Initialize the frontier (e.g., a priority queue) with the starting state.
2. While the frontier is not empty:
    - Choose a node from the frontier based on the search strategy and remove it.
    - If the node contains the goal state, return the solution.
    - Otherwise, expand the node and add its children to the frontier.

### Informed Search Concepts

- **Evaluation Function f(n)f(n):** This function estimates the desirability of considering node nn next. Nodes with a better (typically lower) f(n)f(n) value are considered first. This approach is known as best-first search.
    
- **Path Cost g(n)g(n):** The actual cost from the initial state to node nn.
    
- **Heuristic Function h(n)h(n):** An estimate of the cost from node nn to the nearest goal state.
    - h(n)h(n) is typically based on additional knowledge not inherent in the problem formulation.
    - If h(n1)<h(n2)h(n1​)<h(n2​), it suggests that reaching the goal from n1n1​ is likely cheaper than from n2n2​.
    - By definition, h(goal)=0h(goal)=0.
- **Evaluation Function Variants:**
    - Uniform-Cost Search (UCS): f(n)=g(n)f(n)=g(n). It prioritizes nodes with the lowest path cost from the start. It is goal-directed in terms of cost but not necessarily towards the goal's location.
    - Greedy Best-First Search (GBFS): f(n)=h(n)f(n)=h(n). It prioritizes nodes that are estimated to be closest to the goal, ignoring the cost to reach them.
    - A* Search: f(n)=g(n)+h(n)f(n)=g(n)+h(n). It balances the cost to reach a node with the estimated cost to reach the goal from that node.

### Uniform-Cost Search (UCS)

- **Mechanism:** Orders the frontier by the path cost g(n)g(n). At each step, it selects and expands the node with the lowest total cost from the start.
- **Completeness:** Complete if the step costs are positive.
- **Optimality:** Optimal; it always finds the least-cost solution.
- **Complexity:** Time and space complexity are roughly O(bd)O(bd), where bb is the branching factor and dd is the depth of the shallowest goal.

### Greedy Best-First Search (GBFS)

- **Mechanism:** Orders the frontier using the heuristic function h(n)h(n). It prioritizes nodes that appear closest to the goal, ignoring the actual cost g(n)g(n) to reach them.
- **Completeness:** Not complete. It can get stuck in infinite loops or fail to find a solution if the search space is infinite or very deep.
- **Optimality:** Not optimal. It may find a solution, but not necessarily the least-cost one.
- **Complexity:**
    - Time: O(bm)O(bm), where mm is the maximum depth of the state space. In the worst case, it may explore all nodes up to the maximum depth.
    - Space: O(bm)O(bm), as it may need to store all generated nodes.

### A* Search

- **Mechanism:** Combines UCS and GBFS by using the evaluation function f(n)=g(n)+h(n)f(n)=g(n)+h(n). This aims to find the least-cost path while prioritizing nodes that are both close to the start and estimated to be close to the goal.
    
- **Completeness:** Complete if the step costs are positive.
    
- **Optimality:** Optimal if the heuristic function h(n)h(n) is **admissible**.
    - **Admissible Heuristic:** A heuristic h(n)h(n) is admissible if it never overestimates the true cost to reach the goal from node nn, i.e., h(n)≤h∗(n)h(n)≤h∗(n), where h∗(n)h∗(n) is the true cost.
- **Consistency (Monotonicity):** A stronger condition than admissibility. A heuristic is consistent if for every node nn and every child n′n′ of nn generated by action aa: h(n)≤h(n′)+c(n,a,n′)h(n)≤h(n′)+c(n,a,n′) where c(n,a,n′)c(n,a,n′) is the step cost from nn to n′n′. Consistency implies admissibility. If a heuristic is consistent, A* is guaranteed to find the optimal solution and does not need to re-examine nodes already in the explored set.
    
- **A* Versions:**
    - **Tree Version:** Does not check for repeated states. Simpler but less efficient for graphs.
    - **Graph Version:** Keeps track of explored nodes and the frontier. Updates paths if a shorter one to an already visited node is found. This version is guaranteed to be optimal with a consistent heuristic.
- **Complexity:**
    - Time and Space: Depends heavily on the quality of the heuristic. If h(n)=h∗(n)h(n)=h∗(n) (perfect heuristic), A* explores only the optimal path. If h(n)=0h(n)=0, A* behaves like UCS. In general, the number of nodes explored is related to the difference between the estimated cost and the true cost.

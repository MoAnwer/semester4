## Native (Parametric) End-users

**المستخدمين السطحيين أو "المعلميين" (Naive or Parametric End-users)** كأكبر شريحة من جمهور المستخدمين النهائيين الذين يتفاعلون مع أنظمة قواعد البيانات.

إليك التفاصيل المتعلقة بهذه الفئة وكيفية عملها بناءً على المصادر:

### 1. طبيعة التفاعل: "المعاملات الجاهزة"

- **الوظائف المحددة:** لا يقوم هؤلاء المستخدمون بكتابة استعلامات معقدة، بل يستخدمون وظائف محددة مسبقاً وجاهزة تُعرف باسم **"المعاملات الجاهزة" (Canned Transactions)**.
- **إدخال المعاملات:** يقتصر دور المستخدم على تقديم "معاملات" (Parameters) للنظام — مثل رقم الحساب، أو اسم المستفيد، أو المبلغ — ويقوم التطبيق بتنفيذ العملية المطلوبة بناءً عليها [80، 81].

### 2. أمثلة واقعية من المصادر

تتنوع الأمثلة على هؤلاء المستخدمين لتشمل:

- **صرافو البنوك (Bank-tellers):** الذين يجرون عمليات السحب والإيداع طوال نوبة عملهم.
- **موظفو الحجز (Reservation clerks):** مثل موظفي حجز الطيران أو الفنادق.
- **مستخدمو تطبيقات الجوال:** تندرج معظم تفاعلاتنا مع التطبيقات الحديثة تحت هذه الفئة.
- **مستخدمو وسائل التواصل الاجتماعي:** الذين ينشرون المعلومات أو يقرؤونها عبر الواجهات الجاهزة.

### 3. الدعم الفني والبرمجي لهذه الفئة

بما أن هؤلاء المستخدمين لا يمتلكون بالضرورة معرفة تقنية بهيكل قاعدة البيانات، فإن النظام يعتمد على أدوار أخرى لتهيئة بيئتهم [37، 38]:

- **محللو النظم (System Analysts):** يقومون بفهم متطلبات هؤلاء المستخدمين وتصميم "المعاملات الجاهزة" التي تلبي احتياجاتهم.
- **مبرمجو التطبيقات (Application Programmers):** يتولون كتابة الكود البرمجي لهذه المعاملات واختبارها قبل طرحها للاستخدام.
- **واجهات النظام:** يوفر نظام إدارة قواعد البيانات (DBMS) واجهات سهلة الاستخدام (User-friendly interfaces) مخصصة خصيصاً للمستخدمين "المعلميين" لضمان سهولة الوصول.

### 4. التنفيذ التقني

من الناحية التقنية، عندما يقوم المستخدم "المعلمي" بإدخال المعلمات عبر الكمبيوتر أو الجوال، يقوم **معالج قاعدة البيانات في وقت التشغيل (Runtime Database Processor)** باستلام هذه المعلمات وتنفيذ "الخطة التنفيذية" للمعاملة الجاهزة للوصول إلى البيانات الفعلية وتعديلها أو استرجاعها [81، 82].

**باختصار:** المستخدم السطحي/المعلمي هو العمود الفقري للاستخدام اليومي لقواعد البيانات، حيث يتفاعل مع النظام كأداة وظيفية بسيطة من خلال واجهات برمجية صممها الخبراء مسبقاً [35، 78].
